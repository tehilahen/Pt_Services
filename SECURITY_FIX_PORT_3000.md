# תיקון בעיית אבטחה - חשיפת פורט 3000

## הבעיה שזוהתה

ר"צ אבטחת מידע זיהה שהמערכת עובדת גם על פורט 3000, למרות שפורט 443 אמור להיות מספיק.

## הסיבה לבעיה

1. **קובץ `nginx.conf` היה ריק** - זה גרם ל-nginx לא לנתב נכון את התעבורה
2. **חוסר תצורת Reverse Proxy** - ללא תצורה נכונה, ייתכן שה-frontend נחשף ישירות במקום דרך nginx

## הפתרון שיושם

### 1. תצורת Nginx (`nginx.conf`)
- ✅ הוגדרה תצורת Reverse Proxy מלאה
- ✅ ניתוב `/api/*` ל-backend על פורט 5000 (פנימי)
- ✅ ניתוב `/*` ל-frontend על פורט 3000 (פנימי)
- ✅ תמיכה ב-SSL/TLS על פורט 443 בלבד
- ✅ הוספת Security Headers
- ✅ תמיכה ב-WebSocket עבור React

### 2. Dockerfile ל-Nginx
- ✅ נוצר Dockerfile ל-nginx שמעתיק את התצורה

### 3. אימות תצורת Docker Compose
- ✅ וידאנו ש-`docker-compose.yml` משתמש ב-`expose` ולא ב-`ports` עבור frontend
- ✅ זה מבטיח שפורט 3000 זמין רק בתוך רשת Docker, לא מבחוץ

## מה צריך לעשות עכשיו

### 1. בדיקת תצורת Firewall
וודאו שב-firewall/security group:
- ✅ פורט 443 פתוח (HTTPS)
- ❌ פורט 3000 סגור/חסום מבחוץ
- ❌ פורט 5000 סגור/חסום מבחוץ

### 2. הפעלה מחדש של המערכת
```bash
# בנייה מחדש של nginx עם התצורה החדשה
docker-compose build nginx-pt_service

# הפעלה מחדש של כל המערכת
docker-compose down
docker-compose up -d
```

### 3. בדיקת תקינות
1. בדקו שהמערכת עובדת דרך `https://your-domain:443`
2. וודאו שגישה ישירה ל-`http://your-domain:3000` נכשלת/חסומה
3. בדקו את הלוגים של nginx:
   ```bash
   docker logs nginx-pt_service
   ```

## הסבר למה פורט 443 מספיק

פורט 443 הוא הפורט הסטנדרטי ל-HTTPS (HTTP Secure). ה-Nginx Reverse Proxy:
- מקבל את כל הבקשות על פורט 443
- מנתב אותן ל-frontend או backend לפי הנתיב
- מספק SSL/TLS encryption
- מספק Security Headers נוספים

**פורט 3000 הוא פורט פיתוח** - לא אמור להיות חשוף בפרודקשן!

## הערות נוספות

- אם המערכת עדיין נגישה על פורט 3000, ייתכן שיש תצורת firewall או load balancer נוספת שצריך לבדוק
- וודאו שכל הגישה החיצונית עוברת דרך nginx על פורט 443 בלבד
- בדקו את תצורת ה-security groups ב-AWS (אם רלוונטי)
